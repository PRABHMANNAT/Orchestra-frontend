# Continuity Profile Notes

## Painkiller Frame

The profile is centered on captured rationale and gaps, not file detection.

Captured rationale is the banked "why" behind systems a person owns: decisions from PRs, Slack, reviews, and source docs that remain useful if the person is unavailable.

Gaps are the landmines: owned systems where the why is missing or thin, with an explicit ask-now affordance while the person can still answer.

## Supporting Scope

Ownership and file paths are demoted to scope. They explain what areas need capture, but they are not the product value by themselves because git can already approximate touched files and authorship.

## Mock Behavior

- Planned departure mode shows a rationale pack plus a gap checklist before the mock last day.
- Already gone mode shows a successor handover pack using only captured rationale and warning that gaps remain unknown.
- Capture actions reveal a canned checklist. Real implementation would open a Socrates-assisted interview flow and attach the answer back to the rationale trace.

## Insurance

This is insurance installed before someone leaves: the longer Orchestra runs, the richer the rationale bank becomes and the fewer critical systems become unknowable during handoff.
